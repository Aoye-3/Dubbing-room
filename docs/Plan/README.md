# Plan Documents

This folder stores time-sensitive execution plans.

Plans describe a temporary implementation path, sequencing, assumptions,
verification commands, and completion status. They should not be treated as
long-lived product, architecture, API, or runtime specifications.

When a plan is completed, keep only the durable outcomes in the relevant PRD,
technical, app-dev, or ADR document. Future agents should use this folder for
historical context and task sequencing, then verify current behavior against
the code and durable docs.
